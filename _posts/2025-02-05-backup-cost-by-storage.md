## 백업은 보험인데, 보험료가 얼마인지 아시나요

클라우드에서 백업은 필수다. 문제는 "켜두기만 하면 되겠지"라고 생각하고 방치하면 보험료(백업 비용)가 조용히 불어난다는 것이다. EBS 스냅샷, RDS 자동 백업, S3 버전관리 등 각 스토리지마다 백업 과금 방식이 다르고, 보존 기간이 길어질수록 비용이 누적된다. 백업을 왜 하는지(복구 시간, 보존 요건)를 명확히 하고, 그에 맞는 비용 구조를 이해해야 불필요한 지출을 막을 수 있다.

## EBS 스냅샷

EBS 스냅샷은 가장 흔한 백업 방식이다. 서울 리전 기준 GB-월당 $0.05가 부과된다.

첫 스냅샷은 볼륨 전체(사용된 블록)를 저장하지만, 이후부터는 증분(Incremental) 방식으로 변경된 블록만 저장한다. 500GB EBS 볼륨에서 일일 변경량이 5%라면:
- 첫 스냅샷: 500GB × $0.05 = $25/월
- 이후 일일 스냅샷: 25GB × $0.05 = $1.25/월 (각 스냅샷)
- 30일 보존 시 총량: 500GB + (25GB × 29) = 약 1,225GB → $61.25/월

증분 방식 덕분에 전체 백업보다 훨씬 저렴하지만, 스냅샷 수가 쌓이면 비용도 쌓인다. EBS Snapshots Archive를 활용하면 90일 이상 보관하는 스냅샷을 75% 저렴한 $0.0125/GB-월에 저장할 수 있다.

## RDS 자동 백업

RDS는 무료 백업 구간을 제공한다. 프로비저닝된 스토리지 크기만큼의 백업 저장소가 무료다. 예를 들어 100GB RDS 인스턴스라면 100GB까지의 백업이 무료다.

| 항목 | 비용 |
|------|------|
| 무료 구간 | 프로비저닝된 스토리지 크기까지 |
| 초과분 | $0.095/GB-월 (서울) |
| 보존 기간 | 1~35일 (기본 7일) |

보존 기간을 35일로 늘리면 총 백업 용량이 무료 구간을 초과하기 쉽다. 일일 변경량이 10GB인 100GB RDS에서 35일 보존 시: 총 백업량 ≈ 100GB(기본) + 350GB(증분) = 450GB. 초과분 350GB × $0.095 = $33.25/월. 보존 기간을 7일로 줄이면 초과분이 거의 없어진다.

## S3 버전관리

S3 Versioning을 켜두면 객체를 덮어쓰거나 삭제해도 이전 버전이 남는다. 이것도 일종의 백업이지만, 모든 버전에 대해 저장 비용이 부과된다는 점을 잊기 쉽다.

10GB 파일을 매일 업데이트하고 30일간 버전을 유지하면: 10GB × 30 = 300GB의 저장 비용이 발생한다. Standard 기준 300GB × $0.025 = $7.50/월. Lifecycle Policy에서 `NoncurrentVersionExpiration`을 설정해서 오래된 버전을 자동 삭제하거나 Glacier로 전환해야 한다.

## AWS Backup 통합 관리

서비스마다 개별적으로 백업을 관리하면 정책이 파편화된다. AWS Backup은 EBS, RDS, DynamoDB, EFS, S3 등의 백업을 하나의 정책으로 통합 관리할 수 있다.

| 서비스 | AWS Backup 비용 | 비고 |
|--------|---------------|------|
| EBS | $0.05/GB-월 | EBS 스냅샷과 동일 |
| RDS | $0.095/GB-월 | 무료 구간 별도 |
| DynamoDB | $0.114/GB-월 (웜), $0.012/GB-월 (콜드) | On-Demand 백업 |
| EFS | $0.05/GB-월 | 증분 방식 |
| S3 | $0.05/GB-월 | 별도 비용 |

AWS Backup 자체에 추가 비용은 없고, 각 서비스의 백업 비용만 부과된다. 핵심 가치는 보존 정책(Retention Policy)을 중앙에서 일괄 관리할 수 있다는 점이다.

## 보존 기간에 따른 비용 누적

백업 비용의 가장 큰 변수는 보존 기간이다. "일단 오래 보관하자"는 접근이 비용을 키운다.

| 보존 기간 | EBS 500GB (일일 5% 변경) | RDS 100GB (일일 10GB 변경) |
|----------|------------------------|-----------------------|
| 7일 | ~$30/월 | 무료 구간 내 |
| 30일 | ~$61/월 | ~$24/월 |
| 90일 | ~$144/월 | ~$76/월 |
| 365일 | ~$480/월 | ~$323/월 |

실제 복구가 필요한 시나리오를 정의하고, 그에 맞는 최소 보존 기간을 설정하는 것이 핵심이다. 대부분의 운영 환경에서 7~30일이면 충분하고, 규정 요구가 있는 경우에만 장기 보존(Archive 티어)을 적용하면 된다.

## 증분 백업의 비용 이점

증분 백업은 변경된 데이터만 저장하므로, 전체 백업 대비 저장 공간을 크게 절약한다. EBS 스냅샷, RDS 백업, AWS Backup 모두 증분 방식을 사용한다. 중요한 점은 개별 스냅샷을 삭제해도 다른 스냅샷이 참조하는 블록은 유지된다는 것이다. 즉, 중간 스냅샷을 삭제해도 연쇄 삭제가 발생하지 않아 안전하게 오래된 스냅샷을 정리할 수 있다.

## 마무리

백업 비용은 "저장 단가 × 보존 기간 × 변경량"의 공식으로 결정된다. 서비스별 과금 구조와 무료 구간을 정확히 파악하고, 보존 기간을 최소화하며, AWS Backup으로 정책을 일원화하면 백업 비용을 체계적으로 관리할 수 있다.
