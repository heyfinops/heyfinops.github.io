## AWS에서 가장 이해하기 어려운 비용, 데이터 전송

AWS 청구서를 처음 분석하는 사람들이 공통적으로 하는 말이 있다. "데이터 전송 비용이 왜 이렇게 나오는지 모르겠다." 컴퓨팅이나 스토리지는 단가 × 사용량으로 직관적인데, 데이터 전송은 방향과 경로에 따라 과금 여부가 달라져서 복잡하다. 인바운드는 무료인데 아웃바운드는 유료, 같은 AZ 안에서는 무료인데 AZ를 넘어가면 유료, 같은 리전이면 싼데 리전을 넘어가면 비싸다. 이 구조를 한번 정리해두면 청구서의 미스터리가 상당 부분 풀린다.

## 데이터 전송 과금의 기본 구조

AWS 데이터 전송 비용은 "어디에서 어디로 가는가"에 따라 결정된다. 핵심 원칙은 이렇다.

- **인터넷 → AWS (Inbound)**: 무료
- **AWS → 인터넷 (Outbound, DTO)**: 유료 (GB당 $0.09~$0.12, 리전별 상이)
- **같은 AZ 내 (private IP 사용)**: 무료
- **다른 AZ 간 (Cross-AZ)**: 양방향 각 GB당 $0.01
- **다른 리전 간 (Cross-Region)**: GB당 $0.02 (리전 조합에 따라 상이)

예를 들어 서울 리전에서 인터넷으로 100GB를 내보내면 약 $12.06이 발생한다(처음 10TB 구간 GB당 $0.126, ap-northeast-2 기준). 반면 같은 리전 내 다른 AZ로 100GB를 보내면 보내는 쪽 $1 + 받는 쪽 $1 = $2다.

| 경로 | 방향 | 과금 | 단가 예시 (서울 리전) |
|------|------|------|----------------------|
| 인터넷 → AWS | Inbound | 무료 | $0 |
| AWS → 인터넷 | Outbound | 유료 | ~$0.126/GB |
| 같은 AZ (Private IP) | 내부 | 무료 | $0 |
| 다른 AZ (Cross-AZ) | 양방향 | 유료 | $0.01/GB (각 방향) |
| 다른 리전 | 양방향 | 유료 | $0.02/GB |
| S3 → CloudFront | - | 무료 | $0 (Origin Fetch) |

## VPC 피어링과 Transit Gateway

VPC 간 통신도 비용이 발생한다. **VPC 피어링**은 같은 리전 내라면 Cross-AZ 요금($0.01/GB 양방향)만 부과되고, 다른 리전이라면 Cross-Region 전송 요금이 적용된다. 피어링 자체에 대한 별도 시간당 요금은 없다.

**Transit Gateway(TGW)**는 허브 역할을 하는 네트워크 서비스인데, 데이터 처리 요금이 별도로 붙는다. TGW를 통과하는 데이터에 대해 GB당 $0.02가 과금되며(서울 리전 기준), 여기에 Cross-AZ 요금까지 더해질 수 있다. VPC가 2~3개뿐이라면 피어링이 저렴하고, VPC가 많아서 풀메시 연결이 비현실적이라면 TGW가 관리 측면에서 유리하지만 비용은 더 든다.

## 데이터 전송 비용 절감 전략

데이터 전송 비용을 줄이는 실무 전략은 크게 세 가지다.

**첫째, CloudFront 활용.** S3에서 인터넷으로 직접 데이터를 내보내면 DTO 요금이 발생하지만, CloudFront를 경유하면 S3→CloudFront 구간이 무료이고, CloudFront→인터넷 요금이 일반 DTO보다 저렴하다. 서울 리전 기준 EC2 DTO가 $0.126/GB인 반면, CloudFront는 $0.120/GB(10TB 이하)로 약간 저렴하면서 캐시 히트까지 고려하면 실질 절감이 크다.

**둘째, VPC Endpoint 사용.** EC2에서 S3나 DynamoDB에 접근할 때 인터넷 게이트웨이를 거치면 DTO 비용이 발생할 수 있다. Gateway 타입 VPC Endpoint를 사용하면 이 트래픽이 AWS 내부 네트워크로 흐르면서 무료가 된다. Gateway Endpoint 자체에 시간당 비용도 없다.

**셋째, 같은 AZ에 배치.** 서비스 간 통신이 잦은 컴포넌트들을 같은 AZ에 두면 Cross-AZ 비용이 사라진다. 다만 고가용성을 위해 Multi-AZ를 유지해야 하는 프로덕션 환경에서는 트레이드오프가 있으므로, 개발/테스트 환경에서 적극 활용하는 것이 현실적이다.

## 마무리

AWS 데이터 전송 비용은 방향과 경로가 핵심이다. 인바운드 무료, 아웃바운드 유료, AZ를 넘으면 추가 비용이 기본 원칙이다. CloudFront, VPC Endpoint, 동일 AZ 배치 전략으로 상당한 절감이 가능하다. 청구서에서 "Data Transfer" 항목이 눈에 띄기 시작했다면 경로부터 점검해보는 것이 좋다.
