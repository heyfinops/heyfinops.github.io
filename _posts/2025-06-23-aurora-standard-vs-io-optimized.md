## 개요

Amazon Aurora는 클러스터 구성 시 **Aurora Standard**와 **Aurora I/O-Optimized** 두 가지 요금제를 선택할 수 있다. 둘의 핵심 차이는 I/O 비용을 어떻게 지불하느냐에 있다. Standard는 사용한 I/O 요청 건수만큼 과금되고, I/O-Optimized는 I/O 비용이 0원인 대신 인스턴스와 스토리지 단가가 약 30~40% 더 높다. 워크로드 특성에 따라 총 비용이 크게 달라지기 때문에, 선택을 잘못하면 월 수천 달러를 낭비할 수 있다.

## Standard 요금제

Aurora Standard는 기본 요금제로, 인스턴스 비용 + 스토리지 비용 + I/O 요청 비용(백만 건당 $0.20)으로 구성된다. I/O 사용량이 적은 워크로드에서는 가장 경제적인 선택이다. 하지만 읽기/쓰기가 많은 OLTP 워크로드에서는 I/O 비용이 전체 청구서의 50% 이상을 차지하는 경우도 흔하다. 비용 예측이 어렵고, 트래픽 급증 시 청구서가 급격히 올라가는 단점이 있다.

## I/O-Optimized 요금제

2023년 5월에 출시된 I/O-Optimized는 I/O 요청에 대한 별도 과금이 없다. 대신 인스턴스 비용이 약 30%, 스토리지 비용이 약 2.5배 높다. AWS 공식 가이드에 따르면, I/O 비용이 전체 Aurora 비용의 25% 이상을 차지하는 경우 I/O-Optimized로 전환하면 비용 절감 효과를 볼 수 있다. AWS Compute Optimizer에서도 I/O-Optimized 전환 추천을 제공하므로 기존 클러스터의 전환 시점을 판단하는 데 활용할 수 있다.

## 비교 요약

| 항목 | Standard | I/O-Optimized |
|------|----------|---------------|
| **I/O 과금** | 백만 건당 $0.20 | 무료 (0원) |
| **인스턴스 단가** | 기본 | 약 30% 할증 |
| **스토리지 단가** | $0.10/GB/월 | $0.225/GB/월 |
| **비용 예측성** | 낮음 (I/O에 따라 변동) | 높음 (고정적) |
| **적합한 워크로드** | 읽기 적은 앱, 개발/스테이징 | OLTP, 읽기 집중, 분석 혼합 |
| **비용 절감 기준** | I/O 비용 < 전체 25% | I/O 비용 ≥ 전체 25% |
| **전환 난이도** | - | 클러스터 수정으로 즉시 전환 가능 |
| **다운타임** | - | 없음 (온라인 전환) |

## 선택 가이드

| 상황 | 추천 요금제 |
|------|-------------|
| 개발/테스트 환경 | Standard |
| 트래픽이 낮은 내부 서비스 | Standard |
| 읽기 레플리카를 많이 쓰는 서비스 | I/O-Optimized |
| 주문/결제 등 OLTP 워크로드 | I/O-Optimized |
| 비용 예측이 중요한 SaaS | I/O-Optimized |
| I/O 비용 비중을 모르겠는 경우 | Cost Explorer에서 확인 후 판단 |

## 마무리

가장 확실한 판단 방법은 AWS Cost Explorer에서 현재 Aurora 클러스터의 I/O 비용 비중을 확인하는 것이다. 전체 Aurora 비용 중 I/O가 25%를 넘으면 I/O-Optimized로의 전환을 적극 검토하자. 전환은 클러스터 설정 변경만으로 가능하고 다운타임도 없으니, 실험적으로 적용해보고 1~2주 비용을 비교하는 것도 좋은 접근이다.
