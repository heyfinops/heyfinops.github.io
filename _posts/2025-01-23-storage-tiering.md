## 모든 데이터가 같은 등급일 필요는 없다

냉장고를 생각해보자. 오늘 먹을 반찬은 냉장실에, 다음 달에 먹을 고기는 냉동실에 넣는다. 당장 쓸 데이터와 나중에 혹시 볼 데이터를 같은 스토리지에 두는 건, 반찬이든 고기든 전부 냉장실에 넣는 것과 같다. AWS 스토리지도 접근 빈도에 따라 "등급(Tier)"이 나뉘어 있다. 자주 접근하는 데이터는 비싼 계층에, 거의 안 보는 데이터는 싼 계층에 놓으면 같은 데이터를 보관하면서도 비용이 크게 줄어든다. 이것이 스토리지 티어링(Tiering) 전략이다.

## S3 Storage Class 개요

S3에는 접근 빈도와 복구 시간에 따라 여러 스토리지 클래스가 존재한다. 핵심은 저장 단가가 내려갈수록 검색 비용과 대기 시간이 올라간다는 트레이드오프다.

| Storage Class | 저장 단가 (GB/월, 서울) | 검색 비용 | 최소 보관 기간 | 사용 시나리오 |
|---------------|------------------------|-----------|---------------|--------------|
| Standard | $0.025 | 없음 | 없음 | 자주 접근하는 데이터 |
| Standard-IA | $0.0138 | GB당 $0.01 | 30일 | 월 1~2회 접근 |
| One Zone-IA | $0.011 | GB당 $0.01 | 30일 | 재생성 가능한 비빈번 데이터 |
| Glacier Instant Retrieval | $0.005 | GB당 $0.03 | 90일 | 분기 1회 접근, 즉시 복구 필요 |
| Glacier Flexible Retrieval | $0.0045 | 건당 + GB당 | 90일 | 연 1~2회, 수분~수시간 대기 가능 |
| Glacier Deep Archive | $0.002 | 건당 + GB당 | 180일 | 거의 접근 안 함, 12시간 대기 가능 |

Standard에서 Deep Archive로 갈수록 저장 단가가 약 12배 저렴해진다. 1TB 데이터를 Standard에 두면 월 $25.60이지만 Deep Archive에 두면 월 $2.05다. 10TB면 연간 $2,826의 차이가 난다.

## EBS 볼륨 타입별 단가

EC2에 붙는 EBS 스토리지도 티어링 개념이 적용된다. 워크로드의 IOPS/처리량 요구에 따라 적절한 볼륨 타입을 선택하면 비용을 줄일 수 있다.

| 볼륨 타입 | 단가 (GB/월, 서울) | 기본 IOPS | 사용 시나리오 |
|-----------|-------------------|-----------|--------------|
| gp3 | $0.096 | 3,000 | 범용 (대부분의 워크로드) |
| gp2 | $0.12 | GB당 3 (최대 16,000) | 범용 (레거시) |
| io2 | $0.148 + IOPS 비용 | 프로비저닝 | 고성능 DB |
| st1 | $0.054 | - | 순차 읽기 대량 처리 |
| sc1 | $0.018 | - | 거의 접근 안 하는 콜드 데이터 |

gp2에서 gp3로 바꾸는 것만으로도 GB당 20% 절감이 되면서 기본 IOPS는 3,000으로 동일하거나 더 나은 경우가 많다. 로그 데이터처럼 순차 접근이 주인 경우 st1으로 내리면 gp3 대비 44% 저렴하다.

## 자동 전환: Lifecycle Policy와 Intelligent-Tiering

데이터를 수동으로 일일이 옮기는 건 현실적이지 않다. 자동화 메커니즘 두 가지를 활용하면 된다.

**S3 Lifecycle Policy**: 규칙을 정의해서 일정 기간이 지난 객체를 자동으로 다른 스토리지 클래스로 전환한다. 예를 들어 "생성 후 30일 경과 → Standard-IA로 전환, 90일 경과 → Glacier로 전환, 365일 경과 → Deep Archive로 전환"을 설정할 수 있다. 접근 패턴이 예측 가능한 데이터에 적합하다.

**S3 Intelligent-Tiering**: 접근 패턴을 자동으로 분석해서 최적 계층으로 옮겨준다. 30일간 접근이 없으면 Infrequent Access 계층으로, 90일 없으면 Archive Instant Access 계층으로 자동 이동한다. 접근 패턴을 예측하기 어려운 데이터에 적합하다. 단, 객체당 월 $0.0025/1,000객체의 모니터링 비용이 붙으므로 소량의 작은 파일이 많으면 오히려 비쌀 수 있다.

| 방식 | 동작 원리 | 적합한 상황 | 비용 |
|------|-----------|-------------|------|
| Lifecycle Policy | 기간 기반 규칙 | 접근 패턴이 예측 가능 | 전환 요청 비용만 |
| Intelligent-Tiering | AI 기반 자동 분류 | 패턴 예측 불가 | 모니터링 비용 추가 |

## 마무리

스토리지 티어링은 "데이터의 온도"에 따라 저장 위치를 달리하는 전략이다. S3는 Standard부터 Deep Archive까지 12배의 단가 차이가 있고, EBS도 gp3/st1/sc1로 용도에 맞게 선택할 수 있다. Lifecycle Policy로 자동 전환을 설정해두면 신경 쓰지 않아도 비용이 점점 최적화된다.
